## Что строим и зачем

Внутренний веб-инструмент для замены Qolio — платного сервиса оценки звонков.

**Проблемы с Qolio:** дорогая лицензия + неудобные отчёты + нет интеграции с Битрикс24.

**Пользователи:** 1–5 человек (руководитель + аналитики), компания Entera.

## Ключевые решения (договорённости)

### Чек-листы
- Структура: Чек-лист → Блоки → Критерии
- Оценка критерия: `yes` / `no` / `na` (Да / Нет / Не применимо) + текстовый комментарий
- Расчёт итога: `weighted` (взвешенный) или `average` (среднее) — задаётся на уровне чек-листа
- Все пользователи могут создавать и редактировать чек-листы (нет разграничения ролей)
- Активных чек-листов 2–3 одновременно; остальные архивируются

### Интеграция с Битрикс24
- Данные о сделках тянутся из MySQL Битрикс24 и кешируются в `deal_cache`
- Таблицы: `b24-entera-bitrix24-ru-deals`, `b24-entera-bitrix24-ru-deal-stages`
- Поиск сделки при создании оценки — по ID или имени оператора

### Структура чек-листа ТВК (10 блоков, 100 баллов)
Ключи данных и веса — из `logic_summary.md`, раздел `GROUPS_CORRECT`.  
**Важно:** использовать `GROUPS_CORRECT`, а не `parse_qolio.py` — там ошибка в группировке.

| Блок | Вес |
|------|-----|
| ЛПР-ность | 5 |
| Боль и детализация | 15 |
| Тех часть | 12 |
| ЛМ (личная мотивация) | 12 |
| КМ (корпоративная/коммерческая мотивация) | 10 |
| Вопросы по согласованию счета | 11 |
| Вопросы о возможных препятствиях и барьерах | 7 |
| ДК (дорожная карта) | 8 |
| Подтверждение времени звонка | 5 |
| Блок возражений | 15 |
| Работа с картой сделки | 0 (техн., не влияет) |

### Аналитика (`/reports`)
3 вкладки:
1. **Общие показатели** — тепловая карта по блокам + динамика по неделям
2. **Корреляция** — связь баллов блоков с результатом сделки
3. **По сотрудникам** — сравнительный анализ операторов

Логика расчётов описана в `logic_summary.md`.

### Форма оценки — UX-решения
- Кнопки Да/Нет/Н/П — pill-форма с цветными контурами (зелёный/красный/серый), заливка при выборе
- `{% block styles %}` добавлен в `base.html` — без него CSS дочерних шаблонов игнорировался
- **Дата звонка** — readonly, заполняется из Битрикс (`DATE_CREATE`) при поиске сделки; без сделки пустая
- **Дата оценки** — readonly, проставляется автоматически при сохранении (`updated_at`); при первом создании = `created_at`
- Кнопки Редактировать/Удалить на странице просмотра оценки — перенесены в шапку (были внизу)

### Список оценок (`/evaluations`)
- Фильтры: Сотрудник (autocomplete из уже оценённых), Отдел, Чек-лист, Результат сделки (с tooltip), Дата с/по
- «Результат сделки» заполняется автоматически из Битрикс — сотрудник не вводит вручную
- Сортировка по клику на заголовок: #, Сотрудник, Отдел, Дата звонка, Итог, Результат — с сохранением фильтров

### Модель Evaluation
- Добавлено поле `updated_at: DateTime nullable` — обновляется при каждом сохранении оценки
- Миграция: `ALTER TABLE evaluations ADD COLUMN updated_at DATETIME`

### Чек-лист — страница редактирования
- Кнопки Редактировать/Удалить/В архив/Опубликовать вынесены явно в шапку (были в дропдауне «Действия»)

### Статус оценки (черновик / опубликована)
- Модель `Evaluation` имеет поле `status: "draft" | "published"` (по умолчанию `published`)
- В форме создания/редактирования — две кнопки: «Опубликовать» и «Сохранить черновик»
- Черновики **не попадают в аналитику** (`fetch_evaluations` фильтрует только `published`)
- На странице просмотра черновика — бейдж «Черновик» + кнопка «Опубликовать»
- При попытке опубликовать оценку с `deal_id`, который уже есть в опубликованной оценке того же чек-листа — автоматически сохраняется как черновик с предупреждением
- В списке оценок: бейдж «черновик» в колонке `#`, фильтр «Статус оценки» (Все / Опубликованные / Черновики)

### Что вне MVP
- Импорт исторических данных из Excel — отложено
- Роли пользователей (admin / reviewer) — отложено

## Файловая структура

```
app/
  main.py          — точка входа FastAPI
  models.py        — SQLAlchemy-модели (User, Checklist, Block, Criterion, Evaluation, EvaluationItem, DealCache)
  analytics.py     — логика отчётов (prep_rows, compute_kpi, compute_tab1/2/3)
  auth.py          — bcrypt-авторизация
  bitrix.py        — интеграция с MySQL Битрикс24
  scoring.py       — подсчёт баллов оценки
  routers/         — маршруты: auth, users, checklists, evaluations, reports, api
  templates/       — Jinja2-шаблоны
scripts/
  create_admin.py  — создание первого пользователя
  seed_tvk.py      — наполнение чек-листа ТВК
logic_summary.md   — полная структура ТВК, формулы, источники данных Qolio
```
